# IRIS deploy manifest — one line per deployed file: repo_path|pi_path|flag
# repo_path is relative to the repo root (this file's own repo is the source).
# pi_path is the absolute live path on Pi4. flag is optional:
#   COSMETIC_DRIFT_OK  — known-benign divergence (documented in SNAPSHOT), don't red-flag content diff, only check RAM==SD
#   SYSTEM_PATH        — outside /home/pi (e.g. /etc/...); still checked RAM vs SD the same way, just a different tree
# Blank lines and lines starting with # are ignored.
# NOTE: systemd unit files (assistant.service, iris-web.service) are NOT
# repo-tracked yet and are out of scope for this manifest (RAM==SD only would
# still be checkable if added later, but there's no repo source to compare).
# Runtime data files (iris_config.json, iris_soundboard.json, ps_config.json,
# servo_config.json) are intentionally excluded -- they're operator-editable
# state, not code deploys, so RAM/SD drift there is expected (WebUI writes)
# and comparing against a repo copy would be meaningless.
